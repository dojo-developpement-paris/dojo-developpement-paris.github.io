Le dojo développement de Paris

Le Dojo Développement de Paris est un lieu où des développeurs se retrouvent régulièrement pour travailler sur un défi de programmation pour s'amuser et pratiquer de façon délibérée afin d'améliorer leurs compétences.

Nous nous concentrons sur le développement en public, le plus souvent en partant de zéro, pendant une durée courte (1 h à 1 h 30). Nous utilisons des langages, des outils et des formats d'exercice variés. Nous considérons une séance d'exercice réussie quand elle se termine dans le temps imparti ET que le public repart capable de répéter l'exercice à la maison.

Nous utilisons Mobilizon pour nous retrouver.

Modes de fonctionnement

  • Mob : 1 typist + les autres naviguent
  • Mob strong-style tournant : les mobbers donnent des idées, le navigator décide, le driver tape ; on change les rôles toutes les X minutes avec mobtime
  • Fishbowl : pareil qu'au-dessus en remplaçant le timer par des chaises
  • Randori : chaque personne prend le clavier pendant 5 minutes
  • Demo : 1 personne fait une démo sur un kata + les autres donnent du feedback

Contraintes

  • TDD : nous faisons du TDD par défaut ; ce n'est pas une obligation
  • TCR : test && commit || revert
  • TCRDD : TCR + TDD

Règles

  • Le but est d'apprendre, si possible en s'amusant ; le but n'est pas de finir
  • Tout le monde peut proposer un sujet
  • Il n'y a pas de questions stupides : si vous ne dites pas que vous ne comprenez pas, on ne peut pas le savoir
  • Le vote se fait via ce site : nombre de points = nombre de sujets divisé par 2, arrondi à l'entier inférieur
